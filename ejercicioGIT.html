Examen Git y Github

Alumn@: AGM

  1. ¿Que es un sistema de control de versiones?

    Los sistemas de control de versiones son herramientas de software que ayudan a los equipos de software a gestionar los cambios en el código fuente a lo largo del tiempo. A medida que los entornos de desarrollo se aceleran, los sistemas de control de versiones ayudan a los equipos de software a trabajar de forma más rápida e inteligente.

  2. ¿Cual es la diferencia entre git y github?

    Git es una herramienta, y GitHub es una plataforma.

  3. ¿Cómo creo una rama en git?

    Para crear una rama en git, usamos el siguiente comando: git branch

  4. ¿Cómo clono un repositorio?

    Podemos clonar un repositorio escribimos primero el comando git clone y luego pegamos el enlace al repostiorio que queremos clonar

  5. ¿Cómo puedo bajar los ultimos cambios de un repositorio?

    Para bajar los últimos cambios de un repositorio usamos el comando git pull

  6. ¿Cómo puedo gener un tag?

    Para generar un tag usamos el comando git tag

  7. Cómo hago una mezcla de ramas?

    Para realizar una mezcla de ramas, usamos el comando git merge

  8. ¿Cómo puedo moverme a un commit especifico?

    Para movernos a un commit específico podemos usar git checkout

  9. ¿Cómo puedo borrar una rama?

    Para borrar una rama, podemos usar git delete si es una rama local, o git push origin -d si es remota
